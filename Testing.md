#### What is a regration testing and sanity testing and smoke testing and retesting 
1. Regration testing - Regreation testing means when we added a new functionality to the build we insure that it does not affect the existing functionality 
e.g If we change the login button functionality then we check the home page loading registration and the login is worling or not 

2. Sanity testing - sanity testing mean when we add or update a new feature or fixed some part of a build the we only check the functionality not a entire build 
e.g If we change the functionality of a login button then we only check the login is working or not 

3. Smoke Testing - Smoke Testing mean a when we get a new Build We check the crutial functionality like the login is working or a home page is loading or not, this is also called as a zero day testing. It insure the build is ready to test. 

4. Integration testing - Integration testing means to combile two or more build and test it together.
